# 设计：Android 悬浮对比篮

Status: Approved

## 背景

BuyWise Android 当前覆盖商品首页、AI 导购、识图演示、商品详情和商品对比。现有对比页会自动对比首页前 2-3 个商品，但用户无法在浏览、导购和识图过程中手动收集心仪商品后进入比价分析。

本设计增加 App 内悬浮对比篮，提供类似桌面宠物的轻量常驻反馈，但不申请系统级悬浮窗权限，不做后台桌宠行为。

## 方案

第一版做 App 内右下角迷你悬浮篮：

- 出现范围：首页、AI 导购、识图结果、商品详情页。
- 隐藏范围：商品对比页。
- 加入入口：所有商品卡片和商品详情头部提供加入/移除对比按钮。
- 容量规则：最多 4 个商品，至少 2 个商品才能开始对比。
- 类别规则：允许跨类别加入；若已选商品包含不同类别，展开态显示软提示。
- 状态规则：仅保存在 ViewModel 进程内状态，不做本地持久化；点击对比后保留篮子，只有用户手动清空。
- 导购上下文：从 AI 导购推荐加入商品时记录当前导购需求，并在开始对比时作为 `user_need` 传给现有对比接口；其他入口不强制提供上下文。
- 动画反馈：加入/移除按钮有轻量缩放或高亮反馈，悬浮篮数量 badge 有弹跳反馈。
- 错误反馈：达到 4 个上限、少于 2 个开始对比、无效商品等使用轻提示，不弹 Dialog。

点击悬浮篮进入展开态，用户可移除单个商品、清空篮子或开始对比。开始对比时复用现有 `CompareScreen` 路由，调用已有 Android repository 的 `compareProducts(productIds, userNeed)`，不新增后端 API。

## 影响

- Android ViewModel：增加对比篮状态、加入/移除、清空、开始对比动作。
- Android 数据模型：增加对比篮 UI 状态。
- Android 商品卡片：增加可选加入/移除对比入口和动画反馈。
- Android 页面：首页、AI 导购、识图、详情传入对比篮状态和操作。
- Android 主入口：在 Scaffold 内容层叠加右下角悬浮篮，对比页隐藏。
- Android 对比页：文案从自动首页对比调整为用户已选商品对比。
- 后端 API：无变更。

## 验证

- 首页、AI 导购、识图结果、详情页都能加入/移除同一商品。
- 已加入状态跨页面一致。
- 最多 4 个商品；少于 2 个商品不能开始对比并给出轻提示。
- 点击悬浮篮可展开、移除、清空和开始对比。
- 点击开始对比后进入现有对比页，并使用篮子中的商品调用对比接口。
- 从 AI 导购加入的商品能带上最近导购需求作为 `user_need`。
- 对比页隐藏悬浮篮。
- 加入/移除时有按钮或卡片反馈，数量 badge 有弹跳反馈。
- Android 构建或 Kotlin 编译检查通过。
